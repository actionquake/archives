Mike's giude in making location files
What are these so called location files?
Well..they are a way for the players to report whre they are on the maps, where they saw the enemy, killed an enemy, and so on.
A location is a cubic area, and when you are within that area, and use %L in your say or say_team message, the server will replace "%L" with the name for that location "cube".
Here is an example, "I'm %L" will be replaced with "I'm at the constuction site"

You can also use %S to report a location, but then it'll say the name of the location, where your crosshair aims. So you can say "Enemy spotted %S" and it will say "Enemy spotted on the bridge" for example.

The "cube" is defined with 6 numbers, the first 3 is a point in the 3d space, that all maps are made in, and it's also the center of each location cube, while the last 3 numbers, are width, length and height from the center of the cube, that will define the boundrys of the location.

The location file, should be named the same as the map file is called , but with aqg as extension. ie. BLOCKWAR.bsp, should have a location file named "BLOCKWAR.aqg". It's important to use same chars as the mapname is made by, meaning, if the mapfile is all written with capitol letters, then that's what the location file should use too.

To help you make these locations, i added a new client command, with 5 diffrent parameters to pass along to it.

aqg_location 1
aqg_location 2
aqg_location calc
aqg_location checkL
aqg_location checkS

"aqg_location 1" marks the first position and "aqg_location 2" the second position of the location cube. (see picture below)
When these 2 corners have been marked, you use "aqg_location calc" to display the 6 numbers, defining the location.

-Tip-
I set "logfile 1" before i do a calculation, to get the numbers saved in qconsole.log (in your action folder) so i then can just do a copy & paste into my location file. I also type the location name i want, in the console so it gets written into the logfile, and making it easier for me to remeber what location cube, the numbers reprecent.


"aqg_location checkL" and "aqg_location checkS" is used to be able to echo the location that you're within, while floating around the game, without a round started.

Also, if you use more than 40 location definitions in your location file, you need to set the server cvar "locationlimit" to a higher number than 40, which is default.
