New for V1.35

You can tell I was bored this week. I remembered a discussion I had with someone about the accuracy of the sniper rifle when running and walking and that. So I've added a new server variable, accurate_sniper, which defaults to 1. When activated (1) the sniper is as it always has been, accurate. When set to 0, the sniper takes on inaccuracy when the user moves.

When running the sniper is as accurate as the pistol when walking. When walking the sniper is as accurate as the pistol when standing. When standing or crouched the sniper is deadly accurate.

I'm hoping this can help combat the massive amount of snipers out there, now that most servers have moved anti-gren. In any case this needs to be enabled to take any effect, so it isn't a default option.

New for V1.3

Just the locations stuff has been added.

This allows users to add the %L bit into their say_team commands to report their current position. This doesn't work like the CTF location system which is pretty crappy for AQ2 tbh, but uses location files. Users can create custom files if they like, (using the in built editmode), but they will need to upload them to the server. The server will use default locations files for you if you don't have a custom one on the server.

For more information on creating your own files, download V1.3 and see the locations.txt file for a better description of the commands.

But trust me guys, this addon is gonna make teamplay a lot more fun. Thanks for the idea Tedium (yes he's getting good at this isn't he?).
